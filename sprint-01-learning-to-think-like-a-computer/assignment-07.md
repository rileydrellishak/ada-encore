# 🧩 Modeling Real-World Data

## 🌱 Why are we doing this?

Programs need to represent the real world.

But the real world is messy.

This assignment is about learning how to take messy information and structure it in a way a program can reliably use.

---

## 📚 Learning Objectives

* Use lists, dictionaries, tuples, and sets appropriately
* Choose data structures intentionally
* Represent real-world entities in code
* Understand tradeoffs between different data types
* Organize data for readability and clarity

## 📝 Assignment Spec

---

### Part 1 — Model a real-world object

Choose one:

* a student
* a recipe
* a playlist
* a shopping cart

Represent it using:

* dictionaries
* lists
* at least one additional structure (tuple or set)

---

### Part 2 — Query the data

Write code that:

* extracts information
* summarizes or transforms data
* navigates nested structures

---

### Part 3 — Explain your choices

Write a short explanation:

* Why did you choose each data structure?
* What tradeoffs did you consider?

---

## ⚠️ Common Pitfalls

* overusing dictionaries everywhere
* ignoring structure clarity
* duplicating data unnecessarily
* choosing types without reasoning

---

## ✅ Acceptance Criteria

* You modeled a real-world entity
* You used multiple data structures intentionally
* You extracted meaningful information from it
* Your structure is readable
* You can explain your design choices

---

## ⭐ Stretch Goals (optional)

* Convert your structure to JSON
* Add nested structures
* Validate assumptions about your data
* Compare two different modeling approaches

---

## ⏱️ Estimated Time

~60–90 minutes

---

## 🧠 Reflection Questions

* Did structuring data feel intuitive or abstract?
* What choices felt obvious vs uncertain?
* Did you reconsider how you originally modeled the object?
* What did this reveal about “real-world complexity”?

---

## 🌿 Encore Notes

* Did you over-structure or under-structure?
* Did you duplicate data unintentionally?
* Did clarity improve after restructuring?

---

## 🧰 Engineering Checklist

* [ ] Data model uses appropriate structures
* [ ] At least one nested structure included
* [ ] You can explain your design decisions
* [ ] Data is readable and intentional
* [ ] No unnecessary complexity
