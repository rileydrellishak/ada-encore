# 🧑‍💻 Before You Call It Done

## 🌱 Why this exists

Working code is not the finish line.

Readable code is.

This checklist helps you simulate a gentle code review from another engineer.

---

## 🧭 Clarity

* [ ] Could someone else understand this without explanation?
* [ ] Are variable names meaningful?
* [ ] Is the intent obvious from reading the code?

---

## 🧱 Structure

* [ ] Is logic broken into small pieces?
* [ ] Does each function do one thing?
* [ ] Is repetition minimized?

---

## 🧪 Correctness

* [ ] Did I test multiple inputs?
* [ ] Did I consider edge cases?
* [ ] Does behavior match expectations?

---

## 🧼 Simplicity

* [ ] Is there any unnecessary complexity?
* [ ] Could this be written more directly?
* [ ] Did I over-engineer anything?

---

## 🐛 Debuggability

* [ ] If this broke tomorrow, could I find the issue quickly?
* [ ] Are intermediate values understandable?

---

## 🌿 Final question

> Would I feel comfortable opening this as a pull request?
