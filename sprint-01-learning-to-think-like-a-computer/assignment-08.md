# 🧱 First Steps into Objects

---

## 🌱 Why are we doing this?

So far, you’ve been organizing logic and data separately.

Objects let you combine them.

This is the first step toward modeling real systems in a structured way.

You don’t need mastery here.

You just need familiarity.

---

## 📚 Learning Objectives

* Understand what a class is
* Create simple objects
* Use attributes and methods
* Recognize when objects help organize code
* Connect data + behavior

## 📝 Assignment Spec

---

### Part 1 — Define a simple class

Create a class representing a simple entity:

Examples:

* `Car`
* `Student`
* `Account`

Include:

* at least 2 attributes
* at least 1 method

---

### Part 2 — Create instances

* create at least 2 objects
* call methods on them
* modify attributes

---

### Part 3 — Explain the model

Write a short explanation:

* what does this class represent?
* why did you choose these attributes?
* what behavior belongs to the object?

---

## ⚠️ Common Pitfalls

* overcomplicating class design
* adding too many methods
* confusing class vs instance
* treating OOP as “required abstraction”

---

## ✅ Acceptance Criteria

* You defined a class
* You created instances
* You used methods and attributes
* Your model is simple and understandable
* You can explain your design clearly
