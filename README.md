# 🪩 Ada Encore

A personal re-run of the Ada Developers Academy curriculum — revisited with experience.

---

## 🌱 What this is

Ada Encore is a structured revisit of foundational software engineering concepts from Ada Developers Academy.

It is not a new curriculum.

It is not a tutorial series.

It is a deliberate second pass through fundamentals — this time as someone who has:

* completed Ada Core
* built full-stack projects
* worked in a software engineering internship
* is preparing for a first full-time engineering role

---

## 🎯 Purpose

The goal of Ada Encore is to rebuild and strengthen:

* clarity of thought
* problem decomposition skills
* debugging confidence
* data modeling intuition
* clean coding habits
* engineering communication

This project is less about learning new concepts and more about:

> seeing old concepts with better judgment

---

## 🧠 Core Philosophy

Ada Encore is guided by one question:

> What would Future Me understand now that Past Me didn’t?

From that follows a few principles:

### 1. Clarity over complexity

Simple, readable solutions are preferred over clever ones.

### 2. Thinking over syntax

Code is treated as a reflection of structured thought, not memorized patterns.

### 3. Repetition is intentional

Revisiting fundamentals is not redundancy — it’s reinforcement with perspective.

### 4. Debugging is normal

Errors are expected, not exceptional.

### 5. Small is powerful

Every assignment is intentionally scoped and self-contained.

---

## 🧱 Structure

The repo is organized by sprint:

```text
ada-encore/
│
├── sprint-01-learning-to-think-like-a-computer/
│   ├── README.md
│   ├── assignment-01.md
│   ├── ...
│   ├── resources/
│   └── side-quests/
│
├── sprint-02-...
└── CONTRIBUTING.md
```

Each sprint includes:

### 📘 README.md

A syllabus + mindset guide for the sprint.

### 📝 Assignments

Small, focused exercises inspired by Ada curriculum structure.

### 🧰 Resources

Reusable tools for:

* debugging workflow
* code review habits

### ⭐ Side Quests

Optional explorations for curiosity and reinforcement.

---

## 🧭 How to use this repo

If you are working through Ada Encore:

1. Start with a sprint README
2. Follow assignments in order
3. Use resources when stuck
4. Treat side quests as optional depth
5. Reflect briefly after each assignment
